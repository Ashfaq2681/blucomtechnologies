The single blog page loads that SEO data through two related flows.

  Runtime Page Flow

  1. Route match:
     frontend/src/App.jsx:196 maps /blog/:slug to SingleBlog.

  2. Slug read:
     frontend/src/pages/blog/SingleBlog.jsx:108 uses useParams() to read:
     top-12-marketing-advertising-conferences-in-2026-you-cant-miss

  3. Preloaded post check:
     frontend/src/pages/blog/SingleBlog.jsx:67 checks window.__PRERENDERED_BLOG_POST__.
     If the prerender script injected the post, the page uses that immediately.

  4. API fetch fallback:
     frontend/src/pages/blog/SingleBlog.jsx:154 runs useEffect().
     If no prerendered post exists, it calls:
     frontend/src/api/blogs.js:200

     getPostBySlug(slug)

     That sends:

     GET http://localhost:4000/api/posts/:slug

  5. Frontend normalization:
     frontend/src/api/blogs.js:10 normalizePost() maps DB/API names like seo_title, seo_description,
     canonical_url, schema_json into React-friendly names like seoTitle, seoDescription, canonicalUrl, schemaJson.
  6. Helmet rendering:
     frontend/src/pages/blog/SingleBlog.jsx:200 uses Helmet to render:
      - <title>
      - meta description
      - keywords
      - robots
      - canonical
      - Open Graph tags
      - Twitter tags
      - JSON-LD structured data

     The important computed values are around frontend/src/pages/blog/SingleBlog.jsx:120: seoTitle,
     seoDescription, metaKeywords, socialTitle, socialDescription, socialImage, canonicalUrl, metaRobots.

  Backend API Flow
  The frontend call hits:

  backend/routes/postRoutes.js:22

  router.get("/posts/:identifier", getPostByIdentifier);

  That calls:

  backend/controllers/postsController.js:423

  getPostByIdentifier()

  For a slug, it uses:

  backend/controllers/postsController.js:154

  findRowsBySlug(slug)

  That searches across post tables using:

  backend/utils/contentTables.js:91

  unionPostSelect()

  The API response is shaped by:

  backend/controllers/postsController.js:61

  mapPost(req, row)

  This is where DB columns become the returned fields:

  - seo_title -> seoTitle
  - seo_description -> seoDescription
  - seo_keywords -> seoKeywords / metaKeywords
  - focus_keyword -> focusKeyword
  - canonical_url -> canonicalUrl
  - meta_robots -> metaRobots
  - social_title -> socialTitle
  - social_description -> socialDescription
  - social_image -> socialImage
  - schema_type -> schemaType
  - schema_json -> schemaJson
  - seo_score -> seoScore
  - seo_status -> seoStatus
  - seo_breakdown -> seoBreakdown

  SEO Score Flow
  The score/status/warnings come from:

  backend/utils/seoQuality.js:335

  computeSeoQuality()

  The scoring checks are defined at:

  backend/utils/seoQuality.js:15

  The “Missing internal links” warning specifically comes from:

  backend/utils/seoQuality.js:59

  hasInternalLink()

  It checks whether the post content contains an internal Blucom link.

  Build-Time Prerender Flow
  The raw returned HTML you described is mainly produced by:

  frontend/scripts/prerender-blog-seo.mjs:935

  prerenderBlogSeo()

  Key methods:

  - frontend/scripts/prerender-blog-seo.mjs:159: gets published slugs.
  - frontend/scripts/prerender-blog-seo.mjs:174: loads the post SEO fields.
  - frontend/scripts/prerender-blog-seo.mjs:240: builds title, description, robots, canonical, OG, Twitter,
    keywords, schema.

  - frontend/scripts/prerender-blog-seo.mjs:675: creates window.__PRERENDERED_BLOG_POST__.
  - frontend/scripts/prerender-blog-seo.mjs:719: creates the actual HTML SEO tags.
  - frontend/scripts/prerender-blog-seo.mjs:769: replaces default head SEO.
  - frontend/scripts/prerender-blog-seo.mjs:783: injects the post payload into the page.
  - frontend/scripts/prerender-blog-seo.mjs:861: writes /blog/{slug}/index.html.

  So, for your URL, the title/meta/canonical/social/schema in returned HTML are coming from stored post SEO
  fields, then being injected by the prerender script. When React hydrates, SingleBlog.jsx reads the same injected
  payload first, then falls back to /api/posts/:slug only if needed.
